## 阶段 1: 基础框架重构 + 设计系统 (F1 基础)

- [x] T1: 建立 CSS 变量设计系统
  - **问题**：当前样式使用 Tailwind CDN + 混合样式，无统一设计系统
  - **修复**：
    - 创建完整 CSS 变量体系（颜色、间距、圆角、阴影、字体）
    - 浅色主题为默认（纯白 #ffffff 背景、#171717 文字）
    - 深色主题可选（#0a0a0a 背景、#ededed 文字）
    - 禁止紫蓝品牌色，使用黑白灰中性配色
    - 移除 Tailwind CDN 依赖
  - **涉及文件**：css/main.css
  - **验证**：页面使用 CSS 变量渲染，浅色主题为默认，无紫蓝色

- [x] T2: 重构页面布局框架
  - **问题**：当前布局在移动端排版混乱，侧边栏交互不完善
  - **修复**：
    - PC 端：左侧固定侧边栏（240px）+ 右侧内容区
    - 移动端：顶部栏 + 汉堡菜单 + 抽屉式侧边栏
    - 侧边栏：logo + 导航菜单 + 底部网关状态/启停按钮
    - 顶栏：页面标题 + 右侧操作（主题切换、官方控制台入口、刷新）
    - 内容区：卡片式布局，自适应宽度
  - **涉及文件**：index.html、css/main.css、js/app.js
  - **验证**：PC 和移动端布局正常，侧边栏展开/收起流畅

- [x] T3: 重构仪表盘状态卡片
  - **问题**："OpenClaw 运行正常 版本: 2026.5.18 · 运行时长: 5分钟 · PID: 9577" 样式太丑，毫无设计感
  - **修复**：
    - 设计状态卡片：左侧状态指示灯（大圆点）+ 右侧信息（状态文字、版本、运行时长）
    - 卡片样式：白色背景、1px 边框、8px 圆角、hover 边框变深
    - 状态指示：运行中（绿色圆点 + "运行中"）、已停止（红色圆点 + "已停止"）
    - 信息分组：版本/运行时长/PID 分行显示，使用 secondary 颜色
    - 移动端：卡片全宽，信息紧凑排列
  - **涉及文件**：index.html、css/main.css
  - **验证**：状态卡片有设计感，PC 和移动端均美观

## 阶段 2: 仪表盘完善 + 趋势图 (F1 完整)

- [ ] T4: 修复 CPU/内存趋势图
  - **问题**：趋势图显示一条直线或中间有断点的虚线，不是真正的趋势曲线
  - **修复**：
    - 使用 ECharts 折线图，smooth: true 平滑曲线
    - 正确采集系统 CPU/内存数据（读取 /proc/stat 和 /proc/meminfo）
    - 后端 /api/monitor/cpu 和 /api/monitor/memory 返回历史数据
    - 保留最近 60 个数据点（10 秒采集一次，10 分钟窗口）
    - 图表自适应主题（浅色/深色配色）
    - 填充区域渐变色（非纯色填充）
  - **涉及文件**：js/app.js、css/main.css、serve.py
  - **验证**：趋势图为平滑曲线，实时更新，无断点

- [ ] T5: 增强日志模块
  - **问题**：日志默认 50 条最多 100 条，全部绿色无区分
  - **修复**：
    - 默认 200 条，最多 1000 条（可配置）
    - 颜色区分：ERROR 红色、WARN 黄色、INFO 蓝色、DEBUG 灰色
    - 关键词高亮：错误、异常、超时、error、exception、timeout、failed 等
    - 级别筛选按钮（全部/ERROR/WARN/INFO/DEBUG）
    - 搜索框（实时过滤）
    - 自动刷新开关
  - **涉及文件**：js/app.js、css/main.css、serve.py
  - **验证**：日志 200 条以上，颜色区分正确，筛选可用

- [ ] T6: 实现首次访问 Token 引导
  - **问题**：未配置 Gateway Token 时无提示，用户不知道需要配置
  - **修复**：
    - 首次访问检测 Token 配置
    - 未配置时自动弹出模态弹窗
    - 弹窗内容：说明需要配置 Gateway Token + 输入框 + 保存按钮
    - 保存后自动刷新页面数据
    - "系统设置"链接可正确跳转
  - **涉及文件**：index.html、js/app.js、css/main.css、serve.py
  - **验证**：清空 Token 后访问自动弹窗，配置后正常显示数据

## 阶段 3: 技能管理 (F2)

- [ ] T8: 实现 Skill 详情弹窗 + 启用/禁用

  - **修复**：

    - 弹窗内容：完整描述、分类、触发词、关联文件列表

  - **验证**：详情弹窗显示完整信息，启用/禁用切换正常

- [ ] T9: 实现 Skill 批量操作 + 删除
  - **问题**：无法批量操作，无法删除 Skill
  - **修复**：
    - 卡片左上角添加复选框
    - 顶部显示"已选 N 个" + 批量操作按钮（启用/禁用/删除）
    - 删除操作二次确认弹窗（显示将删除的 Skill 名称列表）

  - **验证**：勾选多个 Skill 后可批量操作，删除有确认弹窗

- [ ] T10: 实现 Skill AI 优化功能
  - **问题**：无法一键优化 Skill
  - **修复**：
    - Skill 详情弹窗添加"AI 优化"按钮
    - 点击后调用后端 /api/skills/:name/optimize
    - 后端读取 Skill 内容，调用 Gateway API 进行优化
    - 优化结果以 diff 形式展示（原内容 vs 优化后内容）
    - 用户确认后应用修改
    - 优化提示词基于 Skill 规范文档
  - **涉及文件**：js/app.js、serve.py
  - **验证**：点击 AI 优化后显示 diff，确认后 Skill 内容更新
