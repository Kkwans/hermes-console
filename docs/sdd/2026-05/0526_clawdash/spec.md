# SPEC — ClawDash: OpenClaw 增强控制台

## 1. 目标与价值

- **解决什么问题**：OpenClaw 内置控制台 UI 差、中文适配不足、功能繁杂难用；绿联控制台功能太少
- **为什么要做**：需要一个面向中文小白用户的、简洁但功能齐全的控制台
- **预期收益**：用户通过 ClawDash 完成 90% 的日常管理操作，无需进入内置控制台

## 2. 上下文与约束

- **技术栈**：Vue 3 (CDN) + ECharts 5 + 自定义 CSS + Python 3 HTTP Server
- **现有架构**：serve.py 后端代理 + 静态前端（/opt/data/console/）
- **依赖**：Hermes Gateway Internal API (localhost:18643)、本地文件系统、config.yaml
- **约束**：
  - 必须在 Hermes Docker 容器内运行
  - 不修改 Gateway 核心代码
  - 后端代理架构（无 CORS）
  - 默认浅色主题，Vercel/Linear 风格黑白灰配色
  - 禁止紫蓝品牌色
  - 所有界面全中文
  - 移动端完全可用

## 3. 功能需求

### F1: 仪表盘 (Dashboard) — P0

**描述**：系统概览页面，展示关键指标和快速入口

**功能**：
- 状态卡片：OpenClaw 运行状态（带设计感的状态指示，非简陋文字）
- 版本信息、运行时长、PID
- 系统资源监控：CPU/内存/磁盘使用率（进度环或仪表盘图）
- CPU/内存趋势图：**必须是平滑曲线图**，非直线或断点虚线
- 已配置模型：当前模型名 + 提供商 + 切换入口
- 连接渠道：中文名 + 图标 + 连接状态
- 最近日志：默认 200 条，最多 1000 条，按类型着色（INFO 蓝、WARN 黄、ERROR 红）
- 网关状态：启停按钮（左下角），官方控制台入口（右上角）
- 首次访问引导：未配置 Gateway Token 时自动弹出配置弹窗

**验收标准**：
- AC1: 仪表盘状态卡片有设计感，非简陋文字
- AC2: 趋势图为平滑曲线，非直线或断点
- AC3: 日志默认 200 条，按类型着色
- AC4: 未配置 Token 时自动弹出配置弹窗
- AC5: 左下角有启停按钮，右上角有官方控制台入口

### F2: 技能管理 (Skills) — P0

**描述**：管理 OpenClaw Skills 的完整生命周期

**功能**：
- **Skill 列表**：
  - 卡片网格展示，显示名称、描述、分类、来源
  - 区分安装渠道：系统自带、平台下载、用户自创
  - 按分类筛选（devops、creative、research 等）
  - 搜索功能（名称 + 描述）
- **Skill 操作**：
  - 启用/禁用单个 Skill
  - 删除 Skill（确认弹窗二次确认）
  - 批量操作：批量启用、批量禁用、批量删除
- **Skill 详情**：
  - 弹窗展示完整 SKILL.md 内容
  - 分类标签、触发词列表
  - 关联文件列表（references、templates、scripts）
- **AI 优化**：
  - 单个 Skill 一键 AI 优化
  - 优化基于 Skill 规范文档
  - 优化前预览 diff，确认后应用
- **第三方安装**：
  - 搜索第三方 Skill 平台
  - 一键安装，自动适配 OpenClaw 规范
  - 中文介绍（英文原文自动翻译）
- **目录配置**：
  - 可配置多个 Skill 目录
  - 自动识别目录下的 Skill

**验收标准**：
- AC6: Skill 列表区分安装渠道，有筛选和搜索
- AC7: 支持批量操作（启用/禁用/删除）
- AC8: AI 优化功能可用，优化后预览 diff
- AC9: 目录配置可添加/删除多个路径

### F3: 模型管理 (Models) — P0

**描述**：管理 AI 模型提供商和模型配置

**功能**：
- **提供商管理**：
  - 提供商列表（卡片展示：名称、API 地址、模型数量）
  - 添加/编辑/删除提供商
  - API Key 配置（密码框，显示/隐藏切换）
  - Base URL 配置
- **模型配置**：
  - 模型列表（按提供商分组）
  - 添加/编辑/删除模型
  - 模型参数配置：名称、上下文长度、能力标签
  - 思考能力设置：关、低、中、高、自动
  - 设置默认模型（全局默认）
  - 设置兜底模型（默认模型不可用时的回退）
- **模型切换**：
  - 一键切换当前使用的模型
  - 切换前验证模型可用性

**验收标准**：
- AC10: 可以添加/编辑/删除提供商
- AC11: 可以添加/编辑/删除模型
- AC12: 思考能力有 5 级选项
- AC13: 可设置默认模型和兜底模型

### F4: 定时任务 (Cron Jobs) — P0

**描述**：管理定时任务的完整生命周期

**功能**：
- **任务列表**：
  - 卡片展示：名称、调度计划、状态、最后执行时间、下次执行时间
  - 状态标签：运行中（绿）、已暂停（黄）、已停止（灰）
  - 按状态筛选
- **任务操作**：
  - 创建任务（表单弹窗：名称、调度方式、prompt、投递渠道、启用开关）
  - 编辑任务（预填表单）
  - 删除任务（确认弹窗）
  - 立即执行
  - 暂停/恢复
- **执行历史**：
  - 查看任务执行历史（时间、状态、耗时、输出摘要）
  - 执行状态：成功（绿）、失败（红）、超时（黄）
- **中文说明**：
  - 调度方式说明（cron 表达式解释）
  - 各配置项附带 ? 帮助按钮

**验收标准**：
- AC14: 任务列表展示完整信息，有状态筛选
- AC15: 创建/编辑表单有中文说明和帮助按钮
- AC16: 执行历史可查看

## 6. 边界与异常

| 场景 | 处理方式 |
|------|----------|
| Gateway 未运行 | 仪表盘显示离线状态，禁用需要 Gateway 的操作 |
| API 请求失败 | Toast 错误提示，不阻断页面 |
| Token 未配置 | 首次访问弹出配置引导弹窗 |
| 移动端操作 | 所有弹窗全屏适配，按钮增大触摸区域 |
| 大量数据 | 分页加载，虚拟滚动（日志/会话） |
