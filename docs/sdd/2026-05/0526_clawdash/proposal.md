# PROPOSAL — ClawDash: OpenClaw 增强控制台

## 1. 问题陈述

### 现状
OpenClaw（Hermes Agent 的宿主平台）内置了一个 Web 控制台（Dashboard），但存在严重问题：

1. **UI 设计极差**：排版混乱，缺乏设计感，仪表盘状态卡片样式粗糙
2. **中文适配不足**：大量页面为英文，插件翻译效果差，对中文用户不友好
3. **功能过于繁杂**：内置控制台包含大量低频配置项，新手小白难以理解和操作
4. **移动端体验烂**：已配置模型、消息渠道、插件等模块在移动端排版和样式严重变形
5. **功能缺陷**：趋势图为直线/断点虚线、日志条数少(50-100)且无颜色区分、无法添加/修改模型配置、消息渠道与插件状态不一致
6. **交互问题**：未配置 Token 时无引导提示、系统设置链接无跳转功能、网关状态显示异常、WebSocket 连接失败
7. **命名残留**：API 路径包含 "ugreen" 等不应存在的标识

## 2. 目标

构建 **ClawDash** — 一个面向中文用户和小白用户的 OpenClaw 增强控制台：

- **简洁但不简陋**：只保留最常用、最高频的配置项，去掉低频复杂功能
- **全中文界面**：所有文案、提示、说明均为中文，术语附带解释
- **专业级 UI**：参考 Vercel/Linear 设计风格，黑白灰中性配色
- **优秀移动端**：Mobile-first 响应式设计，所有页面移动端可正常使用
- **数据可视化**：ECharts 趋势图、状态卡片、进度环等丰富图表
- **增强功能**：内置 AI Skill 优化、批量操作、一键安装等高级功能

## 3. 核心设计理念

### 3.1 与内置控制台的关系
- ClawDash 和内置控制台共用同一端口（6666）
- 默认访问 6666 进入 ClawDash
- ClawDash 内提供跳转到内置控制台的入口（右上角）
- 内置控制台路径：`/builtin/`

### 3.2 设计原则
1. **做减法**：去掉低频功能，保留高频操作
2. **中文优先**：所有文案中文，关键术语附带 ? 帮助按钮
3. **Vercel 风格**：黑白灰配色，无紫蓝品牌色，简洁克制
4. **信息密度适中**：不过于稀疏（绿联的问题）也不过于拥挤（内置控制台的问题）
5. **操作引导**：首次使用有引导，关键操作有确认弹窗

## 4. 技术方案概述

- **前端**：Vue 3 + ECharts + 自定义 CSS 变量体系（无 Tailwind，减少依赖）
- **后端**：Python HTTP Server（增强现有 serve.py）
- **通信**：后端代理 Gateway Internal API（无 CORS）
- **部署**：Docker 容器内运行，端口 6666
- **代码管理**：GitHub 仓库同步

## 6. 风险与依赖

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| Gateway API 不完整 | 部分功能无法实现 | 通过文件系统直接读取配置 |
| 内置控制台端口冲突 | 需要反向代理或端口复用 | 研究 OpenClaw 的端口配置方式 |
| 前端代码量大 | 定时任务单轮无法完成 | 分阶段实现，每轮完成 1-2 个模块 |
| Skill 第三方平台 API | 无法一键安装 | 先实现本地管理，后对接平台 |
