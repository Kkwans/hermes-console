# PROPOSAL — Hermes 控制台全面重构

## 1. 问题陈述

Hermes 控制台（http://192.168.5.110:8089）当前存在严重的设计和功能缺陷，导致完全无法正常使用：

### 核心问题
1. **首次加载异常**：PC端加载极慢，移动端弹出无法操作的空白弹窗（Vue 模板未渲染）
2. **官方控制台链接错误**：指向绿联云而非 Hermes 控制台（http://192.168.5.110:19119/login）
3. **UI 设计全面崩溃**：所有页面的排版、布局、样式均存在严重问题
4. **功能严重缺失**：绝大多数页面只有查看功能，缺少增删改查的基本管理能力
5. **主题适配失败**：浅色模式文字消失、深色模式图表不可见

### 各页面具体问题
- **仪表盘**：已配置模型/连接渠道/最近日志区域无设计
- **会话管理**：ID 展示不全、无编辑功能、详情页白底白字无样式
- **定时任务**：完全空白，无法创建/编辑/删除
- **技能管理**：只能查看，无法搜索安装第三方 Skill
- **插件管理**：只有查看和筛选，无管理功能
- **模型配置**：切换模型需手动输入名称（易出错）、UI 极差
- **消息渠道**：显示未连接（实际已连接）、显示拼音而非中文、无管理功能
- **系统设置**：配置项过少、无说明、UI 差
- **系统监控**：无 Hermes 自身监控数据
- **日志查看**：排版差、关键词无高亮、空间浪费

## 2. 目标

将 Hermes 控制台重构为一个**专业级管理控制台**，参考 Linear/Vercel 等现代 SaaS 产品的设计标准：

- 所有页面具备完整的增删改查功能
- UI 设计达到 2026 年专业水准
- 浅色/深色主题完美适配
- PC/移动端响应式布局
- 所有数据通过后端代理，无 CORS 问题

## 3. 技术方案

### 架构
- **前端**：Vue 3 SPA + ECharts + Tailwind CSS
- **后端**：Python HTTP Server（serve.py）代理 Gateway 内部 API + 本地数据读取
- **设计系统**：Vercel 风格（浅色优先、黑白灰为主、shadow-as-border）
- **字体**：Geist Sans + Geist Mono（Google Fonts CDN）
- **色彩**：纯白背景 #ffffff、近黑文字 #171717、灰色层级 #4d4d4d/#666/#808080/#ebebeb
- **强调色**：仅用功能性颜色（成功绿、错误红、警告黄），不用装饰性品牌色

### 数据流
```
浏览器 → serve.py (8089) → Gateway Internal API (18643)
                         → 本地文件/SQLite
```

## 4. 范围

### 包含
- 全部 10 个页面的 UI 重构
- 全部页面的功能补全（CRUD）
- 后端 API 补全
- 主题适配（浅色/深色）
- 响应式布局

### 不包含
- Gateway 核心逻辑修改
- 新功能开发（如 AI 对话等）
- 第三方集成（除已有飞书外）

## 5. 风险

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 后端 API 能力不足 | 部分功能无法实现 | 优先实现能做的，记录限制 |
| 浏览器兼容性 | 部分 CSS 特性不支持 | 使用成熟方案，避免实验性特性 |
| 性能问题 | 大量数据渲染卡顿 | 分页、虚拟滚动、懒加载 |

- 阶段 3（辅助页面 + 优化）：2-3 轮定时任务
